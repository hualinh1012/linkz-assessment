## Purpose

This document lists every environment variable required to run the platform, their source in each environment, and the Docker Compose setup for local development.

## 5.1 Environment Variable Reference

### Application

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `NODE_ENV` | Runtime environment | `production` | ECS task definition |
| `AUTH_URL` | Canonical URL of the Next.js app | `https://linkz.example.com` | ECS task definition |
| `NEXT_PUBLIC_APP_URL` | Client-side base URL | `https://linkz.example.com` | ECS task definition |

### Database

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `DATABASE_URL` | Prisma connection string | `postgresql://user:pass@host:5432/linkz_db` | AWS Secrets Manager |

### Keycloak / Auth.js

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `KEYCLOAK_ISSUER` | Keycloak realm issuer URL (Auth.js provider) | `https://linkz.example.com/auth/realms/linkz` | ECS task definition |
| `KEYCLOAK_CLIENT_ID` | OIDC client ID | `linkz-web` | ECS task definition |
| `KEYCLOAK_CLIENT_SECRET` | OIDC client secret | `<uuid>` | AWS Secrets Manager |
| `AUTH_SECRET` | Auth.js session-cookie encryption secret | `<base64 32 bytes>` | AWS Secrets Manager |
| `AUTH_TRUST_HOST` | Trust the host header (set `true` behind ALB / in dev) | `true` | ECS task definition |

Auth.js verifies the Keycloak ID token against JWKS internally during the login handshake, so no separate `KEYCLOAK_JWKS_URI` is required.

### AXS Checkout

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `AXS_CLIENT_ID` | AXS-issued client ID — identifies the merchant; embedded in the JWE `kid` field and `clientId` payload | `<axs-client-id>` | AWS Secrets Manager |
| `AXS_API_SECRET` | Shared secret for Payment Page JWE signing and inbound webhook JWE decryption | `<hex>` | AWS Secrets Manager |
| `AXS_CHECKOUT_URL` | AXS Payment Page base URL | `https://checkout.svc.pay.axs.com.sg` | ECS task definition |
| `AXS_API_BASE_URL` | AXS REST API base URL | `https://api.svc.axsasia.com` | ECS task definition |

### Cache

There is **no cache layer at this stage**, so no cache-related environment variables exist (see [SAD/2.3. ADR - Caching](../SAD/2.3.%20ADR%20-%20Caching.md)). When a Redis cache is introduced later, `CACHE_PROVIDER` (`redis`) and `REDIS_URL` will be added here.

### Scheduled Jobs

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `CRON_SECRET` | Secret header value guarding `GET /api/public/cron/cleanup-expired` via `x-cron-secret` | `<random-string>` | AWS Secrets Manager |

### Testing (non-production only)

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `TEST_RESET_TOKEN` | Secret header value guarding `POST /api/public/test/reset-seats` via `X-Test-Token`; must not be set in production | `<random-string>` | `.env.local` / Staging Secrets Manager |

### Keycloak Service (Keycloak ECS task only)

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `KC_DB` | Keycloak database type | `postgres` | ECS task definition |
| `KC_DB_URL` | Keycloak database JDBC URL | `jdbc:postgresql://host:5432/keycloak_db` | AWS Secrets Manager |

| `KC_DB_PASSWORD` | Keycloak DB password | `<secret>` | AWS Secrets Manager |

## 5.2 AWS Secrets Manager Paths

All secrets are stored under a consistent path convention: `/<environment>/<service>/<key>`.

| Secret Path | Value |
|-------------|-------|
| `/production/app/database-url` | Full PostgreSQL connection string |
| `/production/app/keycloak-client-secret` | Keycloak client secret |
| `/production/app/axs-client-id` | AXS Client ID |
| `/production/app/axs-api-secret` | AXS shared secret (Payment Page signing + webhook decryption) |
| `/production/keycloak/db-url` | Keycloak JDBC connection string |
| `/production/keycloak/db-password` | Keycloak DB password |
| `/production/keycloak/admin-password` | Keycloak admin console password |
| `/staging/...` | Same structure, staging values |

ECS tasks retrieve secrets at startup using the ECS `secrets` block in the task definition — values are injected as environment variables and never written to disk or logs.

## 5.3 Local Development — Docker Compose

Local development uses Docker Compose to run all dependencies. The Next.js app itself runs on the host machine via `npm run dev`.

```yaml
# docker-compose.yml
version: '3.9'

services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_USER: linkz
      POSTGRES_PASSWORD: linkz_dev
      POSTGRES_DB: linkz_db
    ports:
      - '5432:5432'
    volumes:
      - postgres_data:/var/lib/postgresql/data

  keycloak:
    image: quay.io/keycloak/keycloak:latest
    command: start-dev
    environment:
      KC_DB: postgres
      KC_DB_URL: jdbc:postgresql://postgres:5432/keycloak_db
      KC_DB_USERNAME: linkz
      KC_DB_PASSWORD: linkz_dev
      KEYCLOAK_ADMIN: admin
      KEYCLOAK_ADMIN_PASSWORD: admin
    ports:
      - '8080:8080'
    depends_on:
      - postgres

volumes:
  postgres_data:
```

### Local `.env.local`

```env
NODE_ENV=development
AUTH_URL=http://localhost:3000
NEXT_PUBLIC_APP_URL=http://localhost:3000

DATABASE_URL=postgresql://linkz:linkz_dev@localhost:5432/linkz_db

KEYCLOAK_ISSUER=http://localhost:8080/realms/linkz
KEYCLOAK_CLIENT_ID=linkz-web
KEYCLOAK_CLIENT_SECRET=<from-keycloak-admin-console>

AUTH_SECRET=<npx auth secret>
AUTH_TRUST_HOST=true

AXS_CLIENT_ID=<sandbox-client-id>
AXS_API_SECRET=<sandbox-api-secret>
AXS_CHECKOUT_URL=https://checkout.svc.uat.pay.axs.com.sg
AXS_API_BASE_URL=https://api.svc.uat.axsasia.com

CRON_SECRET=<random-string>
TEST_RESET_TOKEN=<random-string>
```

### Local AXS Webhook

AXS sandbox must be able to reach `POST /api/public/payment/callback`. Use ngrok during local development:

```bash
ngrok http 3000
# Copy the HTTPS forwarding URL
# Set it as the webhook URL in the AXS developer portal for your sandbox merchant
```

## 5.4 First-Run Setup Checklist

| Step | Command / Action |
|------|-----------------|
| Start dependencies | `docker-compose up -d` |
| Install packages | `npm install` |
| Apply DB migrations | `npx prisma migrate dev` |
| Seed 3 seats | `npx prisma db seed` |
| Build Keycloakify theme | `npx keycloakify build` (first time only) |
| Start dev server | `npm run dev` |
| Configure Keycloak | Create `linkz` realm, `linkz-web` client via Keycloak admin at `http://localhost:8080` |

## 5.5 Environment Differences

| Config | Development | Staging | Production |
|--------|-------------|---------|------------|
| AXS endpoint | Sandbox | Sandbox | Live |
| Keycloak | Docker Compose | ECS (1 task) | ECS (2 tasks) |
| PostgreSQL | Docker Compose | RDS `db.t3.small` Single-AZ | RDS `db.t3.medium` Multi-AZ |
| Cache | None | None | None (reads hit PostgreSQL directly) |
| HTTPS | None (localhost) | ACM certificate | ACM certificate |
| Secrets | `.env.local` | AWS Secrets Manager | AWS Secrets Manager |
| Staging schedule | N/A | Mon–Fri 08:00–20:00 SGT | 24/7 |
