## 1.1 Purpose

This document is the entry point for the **General** section of the Technical Specification for the **Linkz Seat Reservation Platform**. It groups the foundational, cross-cutting technical specifications that the implementation must conform to — project architecture, the current (early-stage) architecture, database, API, authentication, payment, and environment configuration.

Each area has a dedicated child document with the concrete schema, contracts, and code-level detail. The Solution Architecture (the "why") lives under [../SAD](../SAD/1.%20High-level%20solution%20design.md); this section is the "how".
