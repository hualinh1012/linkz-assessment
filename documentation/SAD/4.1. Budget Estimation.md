# 4.1. Budget Estimation

**Status:** ESTIMATE  
**Date:** 2026-06-16  
**Region:** `ap-southeast-1` (Singapore)  
**Authors:** HuaHoangLinh

> All figures are estimates based on AWS Singapore region public pricing as of June 2026. Actual costs vary with traffic, data transfer volume, and usage patterns. Use the [AWS Pricing Calculator](https://calculator.aws) for exact projections before provisioning.

---

## Assumptions

| # | Assumption |
|---|-----------|
| 1 | Early-stage traffic — low to moderate request volume |
| 2 | Production runs 24/7; Staging runs business hours only (Mon–Fri, 8am–8pm SGT) to reduce cost |
| 3 | One ALB per environment (path-based routing handles both Next.js and Keycloak) |
| 4 | Two NAT Gateways in Production (one per AZ for HA); one in Staging |
| 5 | RDS Production: Multi-AZ (`db.t3.medium`); Staging: Single-AZ (`db.t3.small`) |
| 6 | Data transfer out estimated at ~10GB/month Production, ~3GB/month Staging |
| 7 | AXS Checkout is an external service — not included (charges transaction fees only). Identity is self-hosted Keycloak, whose infra cost is already in the ECS Fargate compute line above |
| 8 | SSL/TLS via AWS Certificate Manager (ACM) — free |
| 9 | Domain registered through Route 53 or a third-party registrar |

## Cost Optimisation Notes

| Opportunity | Saving | Trade-off |
|-------------|--------|-----------|
| Staging business-hours schedule (already applied) | ~$85/month vs 24/7 | Staging unavailable nights/weekends |
| Use Savings Plans for Production ECS (1-year commit) | ~20–30% off compute | Upfront commitment |
| Single NAT Gateway in Staging (remove HA) | ~$43/month | No AZ redundancy in staging |
| RDS stop/start schedule for Staging DB | ~$8/month (already applied) | DB unavailable outside business hours |
| CloudFront cache tuning | Reduces data transfer cost | Requires cache policy configuration |
